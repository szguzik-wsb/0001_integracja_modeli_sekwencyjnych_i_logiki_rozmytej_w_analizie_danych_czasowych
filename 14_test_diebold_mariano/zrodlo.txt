METODA
================================================================================

Test Diebold-Mariano (DM) do porownania dokladnosci prognoz dwoch modeli.
Stosowany do weryfikacji, czy roznice w metykach (MAPE, RMSE, MAE) miedzy
modelem hybrydowym TCN+Mamdani a baselinami sa istotne statystycznie.

Hipotezy:
  H0: Oba modele sa rownie dobre (E[d_t] = 0, gdzie d_t = e1_t^2 - e2_t^2)
  H1: Modele roznia sie istotnie w dokladnosci prognoz

Statystyka testowa:
  DM = d_mean / sqrt(V/n)
  gdzie d_t = e1_t^2 - e2_t^2 (roznica kwadratow bledow)
  V uwzglednia autokorelacje bledow dla h-krokowej prognozy

Porownania:
  - TCN+Mamdani vs ARIMA (eksperyment 01 vs 05)
  - TCN+Mamdani vs LSTM (eksperyment 02 vs 05)
  - TCN+Mamdani vs TCN (eksperyment 03 vs 05)
  - TCN+Mamdani vs Mamdani (eksperyment 04 vs 05)

Zbiory danych: SP500, WIG20, EURUSD, BTCUSD (lacznie 16 porownan)
Poziom istotnosci: alpha = 0.05

Interpretacja:
  - DM > 0 i p < alpha: TCN+Mamdani istotnie lepszy
  - DM < 0 i p < alpha: baseline istotnie lepszy
  - p >= alpha: brak istotnej roznicy

Diebold i Mariano (1995) zaproponowali ten test jako standardowe narzedzie
do porownywania dokladnosci prognoz. Jest powszechnie stosowany w literaturze
dotyczacej prognozowania szeregow czasowych finansowych.

================================================================================
IMPLEMENTACJA
================================================================================

Jezyk:             Python 3.11+
Statystyka:        scipy.stats (rozklad t-Studenta)
Obliczenia:        NumPy 1.26+, pandas 2.2+
Implementacja DM:  utils.py — funkcja diebold_mariano(e1, e2, h=1)

Funkcja diebold_mariano:
  - Wejscie: e1 (bledy modelu 1), e2 (bledy modelu 2), h (horyzont)
  - d = e1^2 - e2^2
  - Uwzglednia autokowariancje gamma(k) dla k = 0, ..., h-1
  - Statystyka DM ~ t(n-1)
  - Wyjscie: (DM_stat, p_value) — test dwustronny
