METODA
------
Model TCN (Temporal Convolutional Network) — siec konwolucyjna zaprojektowana
do przetwarzania danych sekwencyjnych. Wykorzystuje konwolucje przyczynowe
(causal convolutions) z dylatacjami rosnacymi wykladniczo, co pozwala na
efektywne modelowanie dlugoterminowych zaleznosci bez uzycia rekurencji.
Architektura zawiera polaczenia rezydualne (residual connections) i normalizacje
BatchNorm w kazdym bloku temporalnym.

TCN stanowi kluczowy komponent proponowanego modelu hybrydowego (TCN + Mamdani)
i jednoczesnie baseline do oceny wartosci dodanej integracji z logika rozmyta.

Architektura:
  - Kanaly:              [32, 32, 32] (3 bloki temporalne)
  - Rozmiar jadra:       kernel_size = 3
  - Dylatacje:           [1, 2, 4] (rosna wykladniczo)
  - Polaczenia rezydualne: tak (residual connections)
  - Normalizacja:        BatchNorm po kazdej konwolucji
  - Konwolucje:          przyczynowe (causal) — brak wycieku informacji z przyszlosci
  - Warstwa wyjscia:     Linear na ostatnim kroku czasowym

IMPLEMENTACJA
-------------
Jezyk:           Python 3.14
Framework:       PyTorch 2.11.0
                 https://pytorch.org/
Implementacja:   wlasna (na podstawie architektury z Bai et al., 2018)
Dane wejsciowe:  znormalizowany szereg cen zamkniecia (Close)
Normalizacja:    MinMaxScaler (zakres [0, 1])

Parametry implementacyjne:
  - input_size = 1
  - num_channels = [32, 32, 32]
  - kernel_size = 3
  - dilations = [1, 2, 4]
  - dropout = 0.2
  - batch_first = True
  - optimizer = Adam(lr=0.001)
  - loss = MSELoss
  - early_stopping_patience = 15
  - lookback_window = 30
  - forecast_horizon = 1

Pole recepcyjne (receptive field):
  R = 2 * (kernel_size - 1) * sum(dilations) + 1
  R = 2 * (3 - 1) * (1 + 2 + 4) + 1 = 29
  Przy lookback=30 model obejmuje cale okno wejsciowe.
